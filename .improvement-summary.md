# Self-Improvement Summary (Iteration 3)

## 1. SQLite Schema Indexes (`lib/src/learning/schema.ts`)
- **Added missing database indexes** for query performance:
  - `idx_findings_created_at` ‚Äî speeds up time-ordered queries on findings
  - `idx_feedback_pr_number` ‚Äî accelerates feedback lookups by PR number
  - `idx_feedback_signal_type` ‚Äî optimizes false-positive rate aggregation queries
  - `idx_patterns_frequency` ‚Äî enables efficient ordering by frequency (DESC)
  - `idx_patterns_key` ‚Äî speeds up pattern lookups by key

## 2. LearningStore Cascade Delete (`lib/src/learning/store.ts`)
- **`deleteFindings()` now wrapped in a transaction** that first deletes related `feedback` rows before removing `findings` rows. This prevents foreign key violations (when SQLite `foreign_keys = ON`) and ensures consistent cleanup.
- Method signature unchanged ‚Äî fully backward compatible.

## 3. EventBus Subscriber Lifecycle (`lib/src/event-bus/bus.ts`)
- **Added `unregister(name: string): boolean`** method that removes a subscriber from all event-type mappings and cleans up its health tracking entry. Returns `true` if a subscriber was found and removed.
- Essential for long-running processes (App mode) where subscribers are dynamically created and disposed.

## 4. Library Detection Accuracy (`lib/src/engine.ts`)
- **Rewrote `detectLibraries()`** with more precise heuristics:
  - Separate Next.js detection from React (looks for `/pages/`, `/app/`, `next.config.*`)
  - Express detection uses `/routes/`, `/middleware/`, `router.*`, `/controllers/`, `.module.ts`
  - Zod detection uses `.schema.ts`, `/schemas/`, `/validators/`, `validation.ts`
  - Added detection for Vue, Svelte, GraphQL, and Vitest
  - Filters out `package.json` and lock files to avoid false positives
- **`detectLibrariesFromDir()`** unchanged (kept as simple directory-based fallback).

## 5. Retry with AbortSignal (`lib/src/utils/retry.ts`)
- **Added optional `signal?: AbortSignal` to `RetryOptions`** for cancellation support.
- `withRetry()` now checks `signal?.aborted` before each attempt and throws `DOMException` ('AbortError') if aborted.
- Fully backward compatible ‚Äî callers that don't pass a signal see no change.

## 6. GitHub API Rate Limit Awareness (`lib/src/utils/github.ts`)
- **Added `checkRateLimit()`** private method called after every API response.
- Warns via `core.warning()` when `X-RateLimit-Remaining` drops below 50.
- Warns once per instance on 429 responses with a `Retry-After` header.
- Uses `pendingWarned` flag to avoid spamming the log with repeated rate-limit warnings.

## 7. JSON DB Multi-Row Insert Support (`lib/src/learning/json-db.ts`)
- **Fixed `run()` method** for findings and feedback tables to handle multi-row INSERT statements (used by `recordFindings()` and `recordFeedbackBatch()`).
- Previously only processed the first row's parameters ‚Äî now iterates over row-sized groups.
