# Improvement Summary

## 1. Circuit Breaker Event Hooks

**Files changed:**
- `lib/src/utils/circuit-breaker.ts` — Added `onOpen`, `onClose`, and `onHalfOpen` optional callbacks to `CircuitBreakerOptions`
- `lib/tests/circuit-breaker.test.ts` — Added 4 new tests covering all callback invocations

**What:** Extended the `CircuitBreaker` class with programmable event hooks that fire on state transitions. The new `onOpen`, `onClose`, and `onHalfOpen` callbacks receive the current `failureCount` and `successCount` from `getMetrics()`, enabling integrations like metrics collection, alerting, or dynamic rate limiting.

**Why:** Previously, state transitions were only observable via `core.info`/`core.warning` log messages. Callers had no programmatic way to react to circuit state changes. These hooks enable monitoring integrations without changing the class's core behavior.

**Type fix:** Also fixed the `DEFAULT_OPTIONS` type from `Required<CircuitBreakerOptions>` to a derived intersection type that keeps callbacks optional, since defaults should not require callback values.
