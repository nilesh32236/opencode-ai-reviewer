# Self-Improvement Summary (Iteration 2)

## 1. Circuit Breaker Pattern (`lib/src/utils/circuit-breaker.ts`)
- **New file** — Added `CircuitBreaker` class implementing the circuit breaker pattern with three states: `CLOSED`, `OPEN`, `HALF_OPEN`.
- Configurable `failureThreshold` (default: 5), `successThreshold` (default: 2), and `cooldownMs` (default: 30s).
- Complements the existing `withRetry()` utility: use the circuit breaker to prevent *any* calls when a service is consistently failing, and `withRetry` for transient failures.
- Exported from `lib/src/index.ts` for use across all packages.

## 2. Structured Logger Utility (`lib/src/utils/logger.ts`)
- **New file** — Added `Logger` class with log levels (`debug`, `info`, `warn`, `error`) and structured context support (PR number, repo, event type).
- Outputs via `@actions/core` methods (`core.debug`, `core.warning`, `core.error`) in GitHub Actions environments.
- Supports child loggers with merged context via `logger.child()`.
- Configurable default log level via `Logger.setDefaultLevel()`.

## 3. EventBus Subscriber Health Tracking (`lib/src/event-bus/bus.ts`)
- Enhanced `EventBus` to track per-subscriber health metrics: total calls, failed calls, last error message, last event type, and last event timestamp.
- Added `getSubscriberHealth()`, `getFailedSubscribers()`, and `resetHealth()` methods for observability.
- Improved error logging to include PR number and repo context.

## 4. GitHub API Pagination (`lib/src/utils/github.ts`)
- Added `paginate<T>()` private method to `GitHubHelper` that automatically follows GitHub pagination by iterating through pages until results are exhausted or max pages (default: 10) is reached.
- Uses `per_page=100` by default and stops early when a page returns fewer items than requested.
- Useful for `getPR()` file listings, `closeOpenCodePRs()`, and any other list endpoints that might exceed 100 results.

## 5. LearningStore Batch Operations (`lib/src/learning/store.ts`)
- Added `recordFindings()` — batch INSERT of multiple findings inside a single transaction for performance.
- Added `deleteFindings()` — bulk delete all findings for a given PR number.
- Added `getFindingsByType()` — filtered query by finding type (e.g., all 'strength' or 'issue' entries).

## 6. New Audit Prompt: Performance & Efficiency (`.audit-prompts/performance-efficiency.md`)
- Added a new audit category covering: N+1 database queries, missing indexes, batch operation opportunities, API pagination, memory usage, unbounded concurrency, and blocking operations.
- Registered in `.opencode-reviewer.yml`.

## 7. Updated `.agents/AGENTS.md`
- Documented the `CircuitBreaker` utility, `Logger` utility, and pagination helper usage guidelines.

## 8. Exports from `lib/src/index.ts`
- Now exports `CircuitBreaker`, `CircuitState`, `CircuitBreakerOptions`, `Logger`, `LogLevel`, `LogContext`, `SubscriberHealth`, `withRetry`, and `withRetryAndTimeout`.
