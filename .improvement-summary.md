# Self-Improvement Summary (Iteration 4)

## 1. DRY Fallback ReviewResult (`lib/src/engine.ts`)
- **Extracted repeated empty result pattern** into a shared `emptyResult()` function.
- Previously the same 10-field fallback `ReviewResult` object was duplicated verbatim in **4 locations** (two in `reviewPR()`, two in `runAudit()`). Now all four reuse the single `emptyResult()` exported from `lib/src/jsonl-parser.ts`.
- Reduces code duplication and ensures consistent default values across all failure paths.

## 2. CircuitBreaker Race Condition Fix (`lib/src/utils/circuit-breaker.ts`)
- **Separated state transitions from the `getState()` getter** into a dedicated `transitionState()` private method. Previously `getState()` mutated the circuit state as a side effect (OPENãHALF_OPEN transition), which violates the principle of least surprise and can cause subtle concurrency bugs when the getter is called from non-`call()` contexts.
- `getState()` now cleanly returns the current state without mutations. `transitionState()` is called at the beginning of `call()` before the OPEN check.
- All tests updated to use `call()` (real usage pattern) instead of relying on the getter side-effect.

## 3. FeedbackSubscriber Error Handling (`lib/src/learning/feedback-subscriber.ts`)
- **Removed redundant `throw err`** after logging in `handle()`. The EventBus already has its own error handling with circuit-breaker per-subscriber, so re-throwing from the subscriber caused double logging and risked unhandled rejection noise. The subscriber now gracefully degrades by logging the warning and returning ã the EventBus circuit breaker still tracks consecutive failures independently.

## 4. Meta-Review Prompt Enhancement (`lib/src/meta-review/prompts.ts`)
- **Added explicit output file path instruction** to the meta-review prompt. The AI model is now told to write its JSON object output to `.opencode/meta-review-output.jsonl` and verify the file after writing. This matches the pattern used by the review and audit prompts, reducing confusion about where to write results.

## 5. App Subscriber Accurate Review Metrics (`app/src/index.ts`, `app/src/handlers/pr-review.ts`)
- **`handlePRReview()` now returns `ReviewResult | null`** instead of `void`, enabling the calling subscriber to propagate actual review metrics.
- **`ReviewSubscriber` in the app** now publishes the `review.completed` event with real data:
  - `reviewSummary` ã the actual summary text from the review
  - `findingsCount` ã total findings (issues + strengths)
  - `issuesCount` / `strengthsCount` ã actual counts
  - `fileCount` ã unique files with issues
- Previously all metrics were hardcoded to `0`, which made meta-review and learning features ineffective when running in App mode.

## 6. LearningStore Consistent Error Handling (`lib/src/learning/store.ts`)
- **`recordFinding()` now returns `string | null`** instead of throwing on failure. This makes it consistent with `recordFeedback()`, `recordQuality()`, and other methods in the same class that silently log warnings and return a fallback value. Previously it was the only method in `LearningStore` that would propagate database errors to the caller.

## 7. JSONL Parser Validation (`lib/src/jsonl-parser.ts`)
- **Added validation** that strength findings have a non-empty `message` string field (matching the existing validation for issue and summary fields).
- Previously, a strength with an empty message would be silently accepted, leading to confusing empty entries in review output.

## 8. Audit Prompt Consistency (`.audit-prompts/*.md`)
- **Standardized output path instructions** across all 4 audit prompts (`code-quality-conventions.md`, `error-handling-resilience.md`, `security-privacy.md`, `performance-efficiency.md`).
- All now reference `.opencode/audit-{category}.jsonl` as the output file instead of `.audit-output.jsonl`, matching the actual path used by `buildAuditPrompt()` in the prompt builder.
