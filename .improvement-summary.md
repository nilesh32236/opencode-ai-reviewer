# Self-Improvement Summary

## 1. Reusable Retry Utility (`lib/src/utils/retry.ts`)
- **New file** — Created a reusable `withRetry()` utility with exponential backoff, jitter, configurable retry counts, and status code filtering.
- Includes `withRetryAndTimeout()` for timed operations.
- Supports custom retryable HTTP status codes (defaults: 429, 500, 502, 503, 504).
- Handles `Response`-like errors, errors with `.status` property, and non-Error throw values.
- Exported from `lib/src/index.ts` for use across all packages.

## 2. GitHubHelper API Retry Logic (`lib/src/utils/github.ts`)
- Wrapped all GitHub API calls in `withRetry()` to handle transient rate limits (429) and server errors (5xx) with exponential backoff.
- Preserves existing error handling - non-retryable errors (4xx except 429) still throw immediately.

## 3. SQLite Transaction Safety (`lib/src/learning/store.ts`, `lib/src/learning/schema.ts`)
- **`recordPattern()`**: Wrapped the read-then-write SELECT+INSERT/UPDATE in `db.transaction()` to prevent race conditions.
- **`incrementAndCheckMetaReviewInterval()`**: Wrapped SELECT+UPDATE in `db.transaction()` for atomic counter increment.
- **`applyMigrations()`**: Wrapped all CREATE TABLE statements in `db.transaction()` for atomic schema initialization.

## 4. MCP Client Retry Logic (`lib/src/mcp/client.ts`)
- Added `withRetry()` wrapping around MCP server connections with 2 retries and 500ms base delay.
