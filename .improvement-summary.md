# Improvement Summary

## 1. Enhanced Audit Prompts (`.audit-prompts/`)

Made all four audit category prompts more precise and actionable by adding:

- **Concrete code examples** showing "bad" (anti-pattern) vs "good" (correct) patterns, directly referencing code patterns used in the codebase (e.g., SQL injection via string interpolation, N+1 query loops, shell injection via command strings)
- **Specific file references** to help the AI focus on the right code locations (e.g., `lib/src/opencode.ts` for shell injection, `lib/src/learning/store.ts` for SQLite operations)
- **Architecture-specific guidance** for WAL mode, prepared statement caching (LRU eviction), JSON database fallback, EventBus subscriber isolation, and `AbortSignal` propagation
- **Sharper severity guides** with more nuanced categories (e.g., "unreachable catch clauses" as critical, "missing AbortSignal checks" as important)
- **Structured logging guidance** distinguishing `core.warning` vs `core.error`/`core.setFailed` usage

Files modified:
- `.audit-prompts/error-handling-resilience.md`
- `.audit-prompts/security-privacy.md`
- `.audit-prompts/code-quality-conventions.md`
- `.audit-prompts/performance-efficiency.md`

## 2. Added LearningStore Test Coverage

Added 5 new tests to `lib/tests/learning-store.test.ts` covering previously untested methods:

- **`addPromptOverride` lifecycle** — verifies prompt overrides are stored and returned via `getRelevantLessons`
- **`declineRule`** — verifies declined rules are removed from pending results
- **`getFindingMessages`** — verifies message/file retrieval with and without file paths
- **`resetCounter`** — verifies meta-review counter reset to zero
- **`recordFeedbackBatch`** — verifies bulk feedback insertion works correctly

## 3. Fixed Test Artifact Linting

- Added `lib/tests/.test-*` to Biome `files.ignore` and `.gitignore` to prevent test-generated JSON artifacts (`.test-pattern.json`, `.test-feedback.json`) from causing lint failures or being committed
